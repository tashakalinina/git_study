Базовые операции

  1. Создаем новый репозиториий (Create a new repository)
  2. Клонируем свой репозиторий на локальный компьютер (Clone)
  3. Отправляем коммиты (изменения в проекте, отправлятся локально) (Commit)
  4. Отправляем созданные коммиты на удаленный git репозиторий (Push)
  5. Получить данные из удаленного git репозитория (Pull)

Откат изменения кода

Чтобы откатить Commit существует следующие варианты:

  • Discart - удаляет незакомиченные изменения
  • Revert - удаляет неотправленные (Push), но уже закомиченный Commit
  • Revert Hurt - удаляет все неотправленные (Push) закомиченные Commit

Работа с ветками

Branch - отдельная ветка (история раработки) программы

  1. Создаем новую ветку (она копирует все содержание ветки master)
  2. Ведем работу в новой ветки, делая все привычные опперации
  3. Сначала добавляем все изменения (если они были) из мастер в новую ветку - команда Merge
  4. Тестируем
  5. Объеденяем ветку с master и делаем push
  6. Дополнительную ветку можно удалять
